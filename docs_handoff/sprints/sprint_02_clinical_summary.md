# Sprint 2 — Clinical Summary

Goal: Convert structured intake JSON into a physician-ready report.

Report sections:

- Chief complaint
- History of Present Illness
- Symptom timeline
- Medications
- Allergies
- Past medical history
- Family history
- Associated symptoms
- Red flags
- Unresolved questions
- Source transcript references

Doctor actions:

- Approve
- Edit
- Reject
- Request more information

Success Criteria:

- Doctor can read a concise summary instead of raw chat.
- Summary clearly separates patient-stated facts from AI interpretation.
